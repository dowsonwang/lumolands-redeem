# lumolands 会员兑换码 · 管理后台 PRD

> 版本：v1.0  ·  对象：开发/测试  ·  语言：中文 UI

## 1. 产品概述

### 1.1 背景与目的

为运营/客服团队提供兑换码全生命周期管理工具，覆盖：批次生成 → 启用 → 兑换码查询 → 作废/停用 → 数据导出。

本后台与用户兑换前台共享同一套兑换码数据，前台兑换的码在后台立即可查。

### 1.2 用户角色

| 角色 | 权限 |
|---|---|
| 管理员（admin） | 批次管理（生成/启用/停用/作废/导出）、兑换码查询、单个作废 |
| 客服（service） | 仅查询（批次列表只读、兑换码列表只读，无操作按钮） |

### 1.3 访问入口

- 登录页：`/admin/login`
- 后台首页：`/admin`（自动重定向到批次管理）
- 左下角浮动按钮"← 返回兑换页"可回到前台

## 4. 批次管理

路由：`/admin/batches`（后台首页）

### 4.1 列表展示

**列定义**

| 列 | 说明 |
|---|---|
| 批次号 | 系统自动生成，格式 `B + YYYYMMDD + 3位序号` |
| 备注 | 文本展示，超长省略，hover 浮窗显示完整内容 |
| 总数 | 该批次下兑换码总数 |
| 已兑换 | 该批次下已使用的码数 |
| 状态 | 未启用（灰）/ 已启用（绿）/ 已作废（红） |
| 创建人 | 创建批次的账号 |
| 创建时间 | 格式化时间 |
| 操作 | 见 4.3 |

**交互**
- 分页：每页 10 条
- 顶部右侧：`刷新` 按钮（管理员还显示 `新建批次` 按钮）
- 客服角色：操作列显示 `仅查看`

### 4.2 新建批次

**入口**：管理员可见的 `新建批次` 按钮

**弹窗字段**
- 会员时长：固定显示 `12 个月`（不可编辑）
- 数量：数字输入，范围 1 - 50000，必填
- 备注：多行文本，必填，占位提示如 `2026-07 批次，印刷订单 PO-888，发往欧洲仓`
- 底部说明：批次创建后默认为未启用状态，码无法兑换，需手动启用

**提交流程**
1. 校验数量范围、备注非空
2. 约 400ms 响应
3. 生成批次 + 批量生成 N 个不重复兑换码（跨批次去重）
4. 成功提示 `批次 BX 已创建，含 N 个兑换码`
5. 关闭弹窗，列表刷新

**异常**
| 场景 | 处理 |
|---|---|
| 数量为空或超范围 | 表单校验红字提示 |
| 备注为空 | 表单校验红字提示 |
| 生成去重失败（理论极低概率） | 抛错，提示重试 |

### 4.3 批次操作

根据批次状态，操作列显示不同按钮：

| 状态 | 可用操作 |
|---|---|
| 未启用 | 导出 CSV / 启用 / 作废 |
| 已启用 | 导出 CSV / 停用 / 作废 |
| 已作废 | 导出 CSV |

#### 4.3.1 启用批次

- Popconfirm 二次确认：`确认启用该批次？启用后码即可兑换。`
- 确认后状态变为 `已启用`，该批次下所有 `unused` 码可被前台兑换
- Toast：`批次 BX 已启用`

#### 4.3.2 停用批次

- Popconfirm 二次确认：`确认停用该批次？停用后码将无法兑换。`
- 确认后状态变为 `未启用`，该批次下码不可兑换（已兑换的不受影响）
- Toast：`批次 BX 已停用`

#### 4.3.3 作废批次

- Popconfirm 二次确认 + 风险说明：
  - 标题：`确认作废该批次？`
  - 描述：`所有未使用的码将被作废，已兑换的码不受影响。此操作不可撤销。`
  - 确认按钮红色危险样式
- 确认后：
  - 批次状态变为 `已作废`
  - 该批次下所有 `unused` 码状态变为 `voided`
  - 已 `used` 的码不受影响
- Toast：`批次 BX 已作废`

#### 4.3.4 导出 CSV

- 点击下载图标，导出该批次下所有兑换码
- CSV 文件名：`{批次号}_codes.csv`
- CSV 表头：`Code,Status,RedeemedAt,RedeemedEmail`
- 兑换码以 4-4-4-4 分隔格式展示（如 `ABCD-EFGH-JKMN-PQRS`），方便人工阅读
- 文件带 UTF-8 BOM 头，确保 Excel 正确识别中文
- Toast：`已导出 N 个兑换码`

## 5. 兑换码查询

路由：`/admin/codes`

### 5.1 查询条件

顶部查询区，支持组合查询：

| 条件 | 控件 | 说明 |
|---|---|---|
| 兑换码 | 文本输入 | 支持模糊匹配，大写 |
| 兑换邮箱 | 文本输入 | 模糊匹配，大小写不敏感 |
| 所属批次 | 下拉选择 | 选项格式 `批次号 (备注)`，可清空 |
| 状态 | 下拉选择 | 未使用/已使用/已作废，可清空 |

**交互**
- 输入框支持回车触发查询
- 顶部右侧：`查询` + `重置` 按钮
- `查询`：将输入框当前值应用为过滤条件
- `重置`：清空所有输入框 + 过滤条件
- 查询区右侧显示 `共 N 条` 结果计数
- `刷新` 按钮：重新拉取数据

### 5.2 列表展示

**列定义**

| 列 | 说明 |
|---|---|
| 兑换码 | 4-4-4-4 分隔格式，等宽字体 |
| 所属批次 | 批次号 |
| 状态 | 未使用（灰）/ 已使用（绿）/ 已作废（红） |
| 兑换邮箱 | 未兑换显示 `-` |
| 兑换时间 | 未兑换显示 `-` |
| 操作 | 详情 / 作废 |

**交互**
- 分页：每页 10 条
- 横向滚动支持（固定兑换码列在左、操作列在右）
- 客服角色：操作列不显示作废按钮

### 5.3 兑换码操作

#### 5.3.1 查看详情

- 点击 `详情` 弹出 Modal
- 展示信息：兑换码、状态、所属批次、创建时间、兑换邮箱、兑换时间、兑换 IP
- 若为补发码，额外展示补发备注
- 下方展示 `兑换尝试记录` 时间线：
  - 倒序展示该码的所有兑换尝试（含成功/失败）
  - 每条显示：结果（成功绿/失败红）、时间、邮箱、IP
  - 无记录时显示 `暂无兑换尝试记录`

#### 5.3.2 单个作废

- 仅 `unused` 状态的码可作废
- 仅管理员可见
- Popconfirm 二次确认：`确认作废该码？`
- 确认后码状态变为 `voided`
- Toast：`兑换码 XXXX-XXXX-XXXX-XXXX 已作废`

**异常**
| 场景 | 处理 |
|---|---|
| 码已被使用，不可作废 | Toast 错误：`只有未使用的码才能作废` |
| 码已被作废 | 操作按钮不显示 |

## 6. 兑换码生成规则（供参考）

> 注：以下为产品层规则，具体实现由开发决定。

- 长度：16 位
- 字符集：大写字母 + 数字，剔除易混淆字符 `0` `O` `1` `I` `L`（共 31 个合法字符）
- 唯一性：同一批次内不重复，跨批次也不重复（与历史所有码去重）
- 展示格式：4-4-4-4 分隔（如 `ABCD-EFGH-JKMN-PQRS`）
- 输入格式：前台用户可不输入分隔符，系统自动忽略

## 7. 异常情况处理

### 7.1 权限异常

| 场景 | 处理 |
|---|---|
| 客服尝试访问需管理权限的操作 | 操作按钮不渲染（不是点击后报错） |
| 会话过期 | 跳转登录页 |

### 7.2 数据异常

| 场景 | 处理 |
|---|---|
| 列表为空 | 表格显示空状态 |
| 查询无结果 | 表格显示空状态 |
| 批次下无码（理论不应发生） | 导出 CSV 仍可下载，文件仅含表头 |

### 7.3 操作冲突

| 场景 | 处理 |
|---|---|
| 两个管理员同时操作同一批次 | 以最后写入为准，操作后刷新可见最新状态 |
| 作废批次时部分码刚被前台兑换 | 已使用的码不受影响，仅未使用的码被作废 |

## 8. 状态机

### 8.1 批次状态

```
新建 ──▶ 未启用 ──启用──▶ 已启用 ──停用──▶ 未启用
              │                  │
              └──作废──▶ 已作废◀──作废──┘
                         （终态）
```

### 8.2 兑换码状态

```
unused ──前台兑换──▶ used（终态）
  │
  └──作废──▶ voided（终态）
```

注：`used` 和 `voided` 均为终态，不可逆。
